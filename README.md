# Анализатор
Ниже о том, как и какие данные собирает наблюдатель для анализа поведенческого паттерна пользовтелей.

## Агрегация
**Интервалы:**
- `t` - Секундный срез (1000ms)
- `10t` - Десятисекундный срез (10000ms)
- `60t` - Минутный срез (60000ms)
- `t15` - Четвертичный срез (900000ms)
- `t60` - Часовой срез (3600000ms)
Каждый интервал наследует предыдущий, метрики можно рассчитывать только при `T > t`, секундные срезы используются только для связи активного процесса с действием пользователя и фиксирования пиковых значений.

## Метрики
### Виды метрик
#### Тип A: Суммарные временные показатели (t_active_mouse, t_unactive_mouse, t_mouse_click, t_keybd_click)
- На **вход** подается массив временных меток событий 
- **Вычисление:**
    - `t_active_mouse` = суммарное время передвижения курсора (-интервал между последовательными событиями `mouse::move`)
    - `t_unactive_mouse` = `T` - `t_active_mouse`
    - `t_mouse_click` = время между первым и последним кликом мыши (если кликов < 2, то 0)
    - `t_keybd_click` = время между первым и последним нажатием клавиатуры (если нажатий < 2, то 0)
#### Тип B: Агрегированые счетчики (min/max/avg)
- На **вход** подается массив значений, полученный для каждого дочернего интервала `T`
- **Вычисление:**
    - Для каждого дочернего интервала в предоставленном массиве вычисляется значение метрики
    - Преобразованные массивы значений обрабатываются:
        - `min` = минимальное значение в массиве (0 если пустой)
        - `max` = максимальное значение в массиве (0 если пустой)
        - `avg` =  среднее арифметическое значение в массиве (0 если пустой)
- **Применяемые метрики:**
    - `mouse_click_count` (m/m/a кол-во нажатий на мышь)
    - `keybd_click_count` (m/m/a кол-во нажатий на клавиатуру)
    - `mouse_S_h` (m/m/a % расстояния по горизонтали)
    - `mouse_S_v` (m/m/a % расстояния по вертикали)
    - `mouse_S_hv` (m/m/a % векторного соотношения горизонтали к вертикали)
    - `mouse_spd` (m/m/a скорость передвижения мыши)
#### Тип C: Составные метрики (коэффициент рваных движений)
- **Вычисление:**
    - `cut_move_kf` = усредненное отношение максимальной мгновенной скорости к средней скорости за дочерние интервалы
    - `cut_move_kf = (max_speed_avg) / (avg_speed)`, где `max_speed_avg` - максимум из средних скоростей по дочерним интервалам, а `avg_speed` - среднее арифметическое всех скоростей за интервал `T`
- **Допустимые значения:**
    - `cut_move_kf > 2` - Неестественное, рваное движение, вероятно ошибка логирования - игнорируется
    - `cut_move_kf > 1` - Естественное, рваное движение
    - `cut_move_kf <= 1` - Плавное, нормальное движение
## Датчики
### Датчики измерения
1. Позиция мышки - `x, y`
2. Нажатая клавиша мыши - `ЛКМ/ПКМ/СКМ` (+удержание)
3. Нажатие на клавиатуру (без идентификации клавиш) с разделением на типы кнопок - `STD/SPC/DGT/NPD`
4. Список активных процессов (без идентификации имен приложений, вкладок и т.д., только название процесса)

**Типы кнопок:**
1. `STD` - Стандартные клавиши (буквы, символы)
2. `SPC` - Специальные клавиши (Shift, F1, Enter и т.д.)
3. `DGT` - Цифры (0-9)
4. `NPD` - Клавиши с NumPad
> Стоит отметить, что программа не сохраняет какие конкретно клавишы были нажаты, а сохраняет только их тип

### Модели измерения
1. Опросная модель - каждый интервал `t` опрашивает диспетчер задач на 3 процесса, занимающие первенство по потребляемым ресурсам. Собираемые данные - имя процесса (eg. `chrome.exe` или `discord.exe`)
2. Событийная модель - получение данных с мышки, клавиатуры. А именно: перемещение(для мышки), нажатие на клавишы. Создает запись только при действии пользователя.

## Логирование
**Формат записи данных:** - `v1::timestamp::event::type::data`
|Параметр|Описание|
|--------|--------|
|`v1`|Версия данных|
|`timestamp`|Временная отметка в формате `%d.%m.%y-%H:%M:%S.%f`|
|`event`|Определение ивента для корректного распознования типа ивентов|
|`type`|Определение типа ивента для корректного распознования данных|
|`data`|Набор данных, характерный для каждого ивента отдельно|
**Ивенты и их типизация:**
```
events:
    mouse:
        move:       %x:%y         // %x:%y - координаты
        scroll:     %dx:%dy       // %dx:%dy - смещение по скроллу
        press:      %x:%y:%b      // %x:%y - координаты, %b - кнопка (ЛКМ/ПКМ/СКМ)
        release:    %x:%y:%b      // %x:%y - координаты, %b - кнопка (ЛКМ/ПКМ/СКМ)
    keyboard:
        press:      %bt           // %bt - тип кнопки 
        release:    %bt           // %bt - тип кнопки 
    proccess:
        get:        %n:%n:%n      // %n - имя процессов в соответствии по потреблению
```

**Модели получения данных с датчиков:**
Стандартный интервал `t` будет выглядеть так:
```
v1::21.08.2007-13:37:00.0100f::proccess::get::chrome.exe:discord.exe:spotify.exe
v1::21.08.2007-13:37:10.0240f::mouse::move::1488:727
```
